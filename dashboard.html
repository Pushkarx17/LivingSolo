My Thoughts on the App

What's working

The Kitchen tab is the strongest feature right now — category cards, expiry colour coding (red/orange/secondary), quantity controls, swipe-to-delete, and search all work together cleanly. That's a solid foundation to reference when building the other tabs.

Biggest gap

The Dashboard tab just says "Coming soon..." — this is the first thing any user sees and it's empty. A good dashboard that summarises the other three tabs would make the entire app feel 10× more polished. Build this next.

Budget needs a rethink

Right now Budget only tracks fixed monthly expenses (like Spotify, rent). It has no dates on the Expense model, no categories, no spending limit/goal, and no way to record what you actually spent versus what you planned. A date field and a "budget limit" would transform it from a static list into something genuinely useful.

To Do priority labels are confusing

The three priority levels are "High", "Priority", and "None" — the word "Priority" as a middle tier doesn't make sense. Rename to "High", "Medium", "Low" for clarity. Also, tasks have no due dates, which limits how useful the list actually is.

Missing features you advertised

The landing page (index.html) promises habit tracking and widget support, but neither exists in the Swift code yet. Either build them or update the landing page copy so it only promises what's actually there.

Big idea — make tabs talk to each other

The app's real value would come from connecting the tabs: Kitchen items expiring soon show up on the Dashboard; a grocery bill gets logged automatically in Budget; an overdue To Do appears in the dashboard alert strip. Right now each tab is a silo. Cross-tab data flow is what turns a collection of features into a coherent app.

Small but important: currency is hardcoded

The £ symbol is hardcoded in BudgetView.swift. Anyone who doesn't use GBP can't use the app properly. Add a currency setting in AboutView (which is currently almost empty anyway) and use it throughout.
